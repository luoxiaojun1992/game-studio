## 修复 UI-008 测试失败 - Mock Server SSE 格式

**问题**: UI-008 测试失败，`handoffItems.length` 为 0，Agent 没有自动创建交接。

**根本原因**: 
Mock server 返回的 SSE 流式响应中 `tool_calls` 格式不符合 OpenAI API 规范。CodeBuddy CLI 期望的格式是：
1. 第一个 chunk 包含 `tool_calls[].index`, `id`, `type`, `function.name`
2. 第二个 chunk 包含 `tool_calls[].index`, `function.arguments`
3. 最后需要发送 `finish_reason: 'tool_calls'`

但 mock server 之前在一个 chunk 中发送了所有内容，导致 CLI 无法正确解析 tool_calls。

**修复内容**:
`tests/mock-server/codebuddy-sdk-mock-server.mjs`:
- 将 tool_calls 拆分为两个 chunk 发送：
  - 第一个 chunk 发送 `id`, `type`, `function.name`
  - 第二个 chunk 发送 `function.arguments`
- 添加 `finish_reason: 'tool_calls'` 到最后的 chunk
- 确保每个 tool_call 使用正确的 `index` 值

**测试结果**: 所有 8 个 UI 测试用例通过 (100% 覆盖率)。
