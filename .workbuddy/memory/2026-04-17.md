## UI-007 Game Count = 0 根因定位与修复 (2026-04-17)

- **根因1**: `submit_game` mock 缺少 `project_id` 参数，zod schema 的 `default('default')` 导致游戏创建到 `default` 项目而非 `ui-007_xxx` 测试项目
- **根因2**: `StudioPage.tsx` 的 `connectSSE` 中 `if (connectedRef.current) return;` 在 `selectedProjectId` 变化后阻止重连

- **经验**: optional + default 的 schema 参数，mock 必须显式传值；调试靠**三层日志链路**（后端 tools → SSE broadcast → 前端 handleSSE → E2E DOM 断言）

## Docker 构建网络经验 (2026-04-17)
- Alpine CDN 连接被拒时，先确认代理端口是否可达（`nc -z` 测试）
- `~/.docker/config.json` 的 `proxies.default` 配置会被 Docker 构建过程读取，导致构建走代理
- 代理关闭后必须删除 `~/.docker/config.json` 中的 proxies 字段，否则构建失败
- 正确流程：构建时带 proxy 运行，运行时去掉 proxy（`up --build -d`）
