## Star-Office Agent 注册流程重构

### 核心问题
- `/agent-push` API 会检查 agent 是否存在，返回 404 如果不存在
- 本地 `registeredAgents` 缓存与 Star-Office-UI 真实状态可能不一致
- Star-Office-UI 重启后 agent 丢失，但本地缓存仍有记录

### 重构后的同步流程
每次同步（agent状态变化 / supervisor定期检查 / 启动时）：
1. **获取全局锁** - 避免并发注册冲突
2. **查询远程真实状态** - 通过 `GET /agents` API 获取 Star-Office-UI 上所有 agent ID
3. **检查本地缓存 vs 远程** - 如果本地有但远程没有，说明 agent 已丢失，需要重新注册
4. **注册 agent（如需要）** - 调用 `/join-agent`
5. **同步状态** - 调用 `/agent-push`
6. **释放锁**

### 新增功能
- `getRemoteAgentIds()` - 查询 Star-Office-UI 上真实的 agent ID 列表
- `ensureAgentRegisteredAndGetId()` - 验证并注册 agent（带远程验证）
- `syncProjectState()` - 完整同步流程：锁 → 验证 → 注册 → 同步状态 → 释放锁
- push 失败重试：如果 404，重新注册后重试推送
